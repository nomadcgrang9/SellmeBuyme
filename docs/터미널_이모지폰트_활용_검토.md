# 터미널에서 TossFaceFontMac 이모지 폰트 활용 가능성 검토

## 📁 현재 상황

**파일**: `public/fonts/TossFaceFontMac.ttf` (12.7MB)
**목적**: 크롤러 모니터링 대시보드에서 이모지를 시각적으로 표시

---

## ✅ 결론: **부분적으로 가능** (조건부)

### 가능한 방법

#### ✅ **방법 1: Windows Terminal 폰트 설정 (가장 현실적)**

**원리:**
- Windows Terminal은 TrueType 폰트(.ttf) 직접 지원
- 사용자가 터미널 설정에서 폰트를 수동으로 변경 가능
- blessed 같은 TUI 라이브러리는 유니코드만 출력, 실제 렌더링은 터미널이 담당

**장점:**
- ✅ TossFaceFontMac의 풍부한 이모지 표현 가능
- ✅ 한글 + 이모지 혼합 깔끔하게 표시
- ✅ Toss의 감성적인 디자인 활용

**단점:**
- ❌ 사용자가 수동으로 설정해야 함
- ❌ 모든 터미널에서 동작하지 않음 (cmd.exe는 제한적)
- ❌ 자동 적용 불가능

**구현:**
```json
// Windows Terminal 설정 (settings.json)
{
  "profiles": {
    "defaults": {
      "font": {
        "face": "TossFaceFontMac",
        "size": 12
      }
    }
  }
}
```

**설치 방법:**
1. `TossFaceFontMac.ttf` 더블클릭 → "설치" 버튼
2. Windows Terminal 실행 → `Ctrl+,` (설정)
3. 모양 → 글꼴 → "TossFaceFontMac" 선택
4. 저장 후 재시작

---

#### ✅ **방법 2: 기본 유니코드 이모지 사용 (범용적)**

**원리:**
- Windows 10+ 기본 Segoe UI Emoji 폰트 사용
- blessed에서 유니코드 이모지 문자 직접 출력
- TossFaceFontMac 없이도 작동

**장점:**
- ✅ 추가 설정 불필요
- ✅ 모든 Windows 10+ 환경에서 작동
- ✅ 자동으로 적용됨

**단점:**
- ❌ Toss 스타일 이모지 사용 불가
- ❌ 시스템 기본 이모지만 사용 (Microsoft 스타일)

**사용 가능한 유니코드 이모지:**
```javascript
// 상태 아이콘
'✅'  // U+2705 완료
'❌'  // U+274C 실패
'🔄'  // U+1F504 진행중
'⏳'  // U+23F3 대기
'📊'  // U+1F4CA 통계
'📈'  // U+1F4C8 증가
'📋'  // U+1F4CB 목록
'🗺️'  // U+1F5FA 지도
'⏱️'  // U+23F1 타이머
'🎯'  // U+1F3AF 타겟
'🚀'  // U+1F680 로켓
'⚠️'  // U+26A0 경고
```

---

#### ⚠️ **방법 3: 동적 폰트 로딩 (불가능)**

**시도:**
```javascript
// blessed에서 커스텀 폰트 로드?
const screen = blessed.screen({
  font: 'TossFaceFontMac'  // ❌ 작동 안 함
});
```

**이유:**
- blessed는 폰트를 직접 로드하지 않음
- 터미널 앱(Windows Terminal, cmd.exe)이 폰트 렌더링 담당
- Node.js 프로세스가 터미널 폰트를 제어할 수 없음

**결론**: ❌ **불가능**

---

#### ⚠️ **방법 4: Canvas/Electron으로 렌더링 (오버킬)**

**원리:**
- Electron 앱으로 만들어서 HTML Canvas에 폰트 로드
- 터미널이 아닌 별도 GUI 창 표시

**장점:**
- ✅ TossFaceFontMac 완벽 활용
- ✅ 시각적으로 화려함

**단점:**
- ❌ 터미널 느낌 사라짐
- ❌ 구현 복잡도 증가 (Electron 번들링 필요)
- ❌ 리소스 소모 큼

**결론**: ⚠️ **가능하지만 비현실적**

## 🛠️ 실제 구현 예시

### blessed에서 유니코드 이모지 사용

```javascript
import blessed from 'blessed';

const screen = blessed.screen({
  smartCSR: true,
  fullUnicode: true  // ✅ 유니코드 이모지 지원 활성화
});

const header = blessed.box({
  top: 0,
  left: 0,
  width: '100%',
  height: 3,
  content: '📊 35개 크롤러 실행 현황        [2026-01-25 11:42:35]',
  tags: true,
  border: { type: 'line' },
  style: {
    border: { fg: 'cyan' },
    fg: 'white'
  }
});

const regionGrid = blessed.box({
  top: 10,
  left: 2,
  width: '100%-4',
  height: 10,
  content: `
🗺️  지역별 현황

광역시도 (17개)
  ✅ seoul      ✅ busan      ✅ daegu      ✅ incheon
  ✅ gwangju    ✅ daejeon    ✅ ulsan      ✅ sejong
  ✅ gyeonggi   ✅ gangwon    ✅ chungbuk   ✅ chungnam
  ✅ jeonbuk    ✅ jeonnam    ✅ gyeongbuk  ✅ gyeongnam
  ✅ jeju

경기 기초지자체 (18개)
  🔄 yangpyeong ⏳ pocheon    ⏳ yeoncheon  ⏳ dongducheonyangjyu
  ⏳ gapyeong1  ⏳ gapyeong2
  `,
  border: { type: 'line' },
  scrollable: true
});

screen.append(header);
screen.append(regionGrid);
screen.render();
```

**결과:**
- ✅ 이모지가 제대로 표시됨
- ✅ 색상도 적용됨
- ✅ 박스 그리기와 혼용 가능
- ✅ 스크롤 가능

        ↓
