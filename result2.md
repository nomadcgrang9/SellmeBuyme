# UI/UX 개선 코드 변경사항 상세 설명

## 개요

이번 커밋에서는 사용자 경험(UX)을 개선하기 위해 **5개의 새 컴포넌트**를 추가하고, **2개의 기존 컴포넌트**를 수정했습니다.

---

## 1. EmptyState 컴포넌트 (신규)

### 파일 위치
`src/components/common/EmptyState.tsx`

### 이게 뭐야?
검색 결과가 없거나 데이터가 비어있을 때 보여주는 화면입니다.

### 수정 전 (기존)
```
┌─────────────────────────────┐
│                             │
│   표시할 공고가 없습니다      │  ← 단순 텍스트만 표시
│                             │
└─────────────────────────────┘
```

### 수정 후 (신규)
```
┌─────────────────────────────┐
│          🔍                 │  ← 애니메이션 아이콘
│   "서울" 검색 결과가 없어요   │  ← 친근한 메시지
│   다른 검색어로 시도해 보세요  │
│                             │
│   추천 검색어                │
│   [서울] [경기] [기간제교사]  │  ← 클릭 가능한 추천어
│                             │
│      [🔄 필터 초기화]        │  ← 액션 버튼
└─────────────────────────────┘
```

### 주요 기능
1. **4가지 타입의 일러스트레이션**
   - `search`: 검색 결과 없음 (🔍 아이콘)
   - `filter`: 필터 결과 없음 (✨ 아이콘)
   - `location`: 지역 공고 없음 (📍 아이콘)
   - `default`: 기본 빈 상태 (📋 아이콘)

2. **추천 검색어 제안**
   - 서울, 경기, 기간제교사, 방과후강사 등 클릭 가능

3. **액션 버튼**
   - 필터 초기화, 다른 지역 선택 등

## 2. CardSkeleton 컴포넌트 (신규)

### 파일 위치
`src/components/common/CardSkeleton.tsx`

### 이게 뭐야?
데이터가 로딩 중일 때 보여주는 "뼈대" 화면입니다. 실제 카드 모양과 비슷하게 생긴 회색 박스가 깜빡입니다.

### 수정 전 (기존)
```
로딩 중...     ← 또는 아무것도 안 보임
```

### 수정 후 (신규)
```
┌─────────────────────────────┐
│ ▬▬▬▬▬▬▬▬▬▬▬▬▬▬▬▬▬▬▬▬▬▬▬ │  ← 파란색 상단 바
├─────────────────────────────┤
│ [====]            [===]     │  ← 깜빡이는 회색 박스들
│ [================]          │
│ [===========]               │
│ (===) (====)                │
│ 📍 [======]                 │
│ 💰 [====]                   │
│ ⏰ [========]      [D-3]    │
└─────────────────────────────┘
```

### 제공되는 스켈레톤 종류
| 컴포넌트 | 용도 |
|---------|------|
| `CardSkeleton` | 카드 그리드용 (6개 기본) |
| `SingleCardSkeleton` | 단일 카드 로딩 |
| `CompactCardSkeleton` | AI 추천 카드용 |
| `ListSkeleton` | 모바일 리스트용 |
| `MapSkeleton` | 지도 로딩용 (회전하는 원) |

### 타입별 색상
- **공고(job)**: 파란색 (`#68B2FF`)
- **인재(talent)**: 녹색 (`#7db8a3`)
- **체험(experience)**: 노란색 (`#f4c96b`)

## 3. SearchResultSummary 컴포넌트 (신규)

### 파일 위치
`src/components/search/SearchResultSummary.tsx`

### 이게 뭐야?
검색 결과를 요약해서 보여주는 컴포넌트입니다.

### 수정 전 (기존)
```
공고 목록     30개    ← 단순 숫자만
```

### 수정 후 (신규)
```
🔍 "서울" · 학교공고 검색 결과 30개  [서울 ✕] [초등 ✕]  초기화
```

### 주요 기능
1. **검색어 표시**: `"서울"` 형태로 강조
2. **필터 태그**: 적용된 필터를 태그로 표시, 개별 삭제 가능
3. **초기화 버튼**: 전체 필터 한번에 제거

## 4. FilterTags 컴포넌트 (신규)

### 파일 위치
`src/components/filters/FilterTags.tsx`

### 이게 뭐야?
현재 적용된 필터를 태그 형태로 보여주고 삭제할 수 있게 해줍니다.

### 화면 예시
```
🏷️ 필터  [학교급: 초등 ✕]  [과목: 수학 ✕]  [지역: 서울 ✕]  🔄 초기화
```

### 색상 옵션
- `blue`: 파란색 (기본)
- `green`: 녹색
- `purple`: 보라색
- `amber`: 주황색
- `gray`: 회색

## 5. MobileBottomNav 개선 (수정)

### 파일 위치
`src/components/mobile/MobileBottomNav.tsx`

### 이게 뭐야?
모바일 화면 하단의 네비게이션 바입니다.

### 수정 전
```
┌─────────────────────────────────────────────┐
│   🏠      ⇄      ➕      💬      👤        │  ← 아이콘만
└─────────────────────────────────────────────┘
문제: 아이콘만 있어서 무슨 기능인지 모름
```

### 수정 후
```
┌─────────────────────────────────────────────┐
│   🏠      💼      ⊕      💬      👤        │
│   홈     공고    등록    채팅     MY        │  ← 레이블 추가
│   •                                         │  ← 활성 탭 표시
└─────────────────────────────────────────────┘
```

### 변경 내용

| 항목 | 수정 전 | 수정 후 |
|------|--------|--------|
| 레이블 | 없음 | 모든 버튼에 레이블 추가 |
| 토글 버튼 | `⇄` 아이콘만 | 타입별 아이콘+레이블 (공고/인재/체험) |
| 등록 버튼 | 회색 | 파란색 원형 강조 |
| 활성 탭 | 색상 변경만 | 색상 + 하단 점 인디케이터 |
| 애니메이션 | 없음 | 탭 전환 시 스프링 애니메이션 |

### 토글 버튼 상세
탭할 때마다 아이콘과 레이블이 바뀝니다:

```
1번째 탭: 💼 공고 (파란색)
2번째 탭: 👥 인재 (녹색)
3번째 탭: ✨ 체험 (노란색)
→ 다시 1번으로 순환
```

## 6. CardGrid 개선 (수정)

### 파일 위치
`src/components/cards/CardGrid.tsx`

### 이게 뭐야?
카드들을 그리드 형태로 보여주는 컴포넌트입니다.

### 추가된 기능

#### 새로운 Props
```typescript
interface CardGridProps {
  cards: Card[];
  loading?: boolean;           // 🆕 로딩 상태
  searchQuery?: string;        // 🆕 검색어
  onResetFilters?: () => void; // 🆕 필터 초기화
  onSuggestionClick?: () => void; // 🆕 추천어 클릭
  viewType?: 'job' | 'talent' | 'experience'; // 🆕 뷰 타입
}
```

#### 동작 흐름
```
CardGrid 컴포넌트
    │
    ├─ loading = true?
    │   └─→ CardSkeleton 표시 (뼈대 화면)
    │
    ├─ cards.length === 0?
    │   ├─ searchQuery 있음?
    │   │   └─→ SearchEmptyState 표시
    │   │       ("서울" 검색 결과가 없어요)
    │   │
    │   ├─ onResetFilters 있음?
    │   │   └─→ FilterEmptyState 표시
    │   │       (조건에 맞는 공고가 없어요)
    │   │
    │   └─ 기본
    │       └─→ EmptyState 표시
    │           (표시할 학교공고가 없어요)
    │
    └─ cards 있음
        └─→ 기존대로 카드 그리드 표시
```

## 8. 사용 예시

### EmptyState 사용
```tsx
import { SearchEmptyState } from '@/components/common/EmptyState';

// 검색 결과 없을 때
<SearchEmptyState
  query="서울"
  onReset={() => resetFilters()}
  onSuggestionClick={(word) => search(word)}
/>
```

### CardSkeleton 사용
```tsx
import CardSkeleton from '@/components/common/CardSkeleton';

// 로딩 중일 때
{loading ? (
  <CardSkeleton count={6} type="job" />
) : (
  <CardGrid cards={cards} />
)}
```

### CardGrid 새로운 Props 사용
```tsx
<CardGrid
  cards={cards}
  loading={isLoading}           // 🆕
  searchQuery={query}           // 🆕
  onResetFilters={handleReset}  // 🆕
  viewType="job"                // 🆕
/>
```

## 9. 시각적 비교 요약

### 빈 상태 화면
| 수정 전 | 수정 후 |
|--------|--------|
| "표시할 공고가 없습니다" | 🔍 애니메이션 + 친근한 메시지 + 추천 검색어 + 액션 버튼 |

| 수정 전 | 수정 후 |

## 10. 결론

이번 변경으로 다음이 개선됩니다:

1. **빈 상태 UX** 🎯
   - 사용자가 "왜 결과가 없는지" 이해 가능
   - "다음에 뭘 해야 하는지" 안내 (추천 검색어, 필터 초기화)

2. **로딩 경험** ⏳
   - 데이터 로딩 중임을 명확히 인지
   - 앱이 멈춘 게 아님을 확인

3. **모바일 네비게이션** 📱
   - 각 버튼이 무슨 기능인지 한눈에 파악
   - 현재 어떤 탭에 있는지 명확히 표시
