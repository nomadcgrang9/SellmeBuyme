# 지역 검색 로직 개선 계획

## 현황 분석

### 1. 현재 데이터 구조

#### 1.1 DB 스키마 (`job_postings.location`)
- **타입**: `TEXT NOT NULL`
- **저장 값 예시**: `의정부`, `남양주시`, `서울`, `강원`, `춘천시`, `원주시` 등
- **문제점**: 광역시도(경기도)와 기초자치단체(의정부시)가 혼재되어 저장

#### 1.2 크롤러별 location 저장 패턴

| 크롤러 | 저장 패턴 | 예시 |
|--------|----------|------|
| 경기도 | 기초자치단체명 | `의정부`, `남양주시`, `수원시` |
| 서울 | 구 이름 또는 "서울" | `강남구`, `서초구`, `서울` |
| 인천 | 구 이름 | `부평구`, `서구`, `동구` |
| 강원 | 시/군 이름 또는 "강원" | `춘천시`, `원주시`, `강원` |

#### 1.3 검색 필터 옵션 (`filters.ts`)
```typescript
REGION_OPTIONS = [
  '서울 전체',
  '경기도 전체',
  '수원', '고양', '용인', ... // 경기도 기초자치단체만
]
```

### 2. 현재 검색 로직 (`queries.ts`)

```typescript
// 지역 필터 적용 부분 (3310-3316행)
if (hasFilterValue(filters.region, DEFAULT_REGION)) {
  const regionPattern = buildIlikePattern(filters.region);
  query = query.or(`location.ilike.${regionPattern},location.ilike.%${filters.region}시%`);
}
```

**문제점**:
1. `경기도 전체` 검색 시 → `%경기도%` 패턴으로 검색
2. DB에 `의정부`, `남양주시` 등으로 저장된 데이터는 매칭 안 됨
3. **광역시도 → 기초자치단체 계층 관계가 반영되지 않음**

## 문제점 상세

### 핵심 문제: 지역 계층 구조 미반영

```
[사용자 기대]
"경기도 전체" 검색 → 경기도 내 모든 시/군 공고 표시
  - 의정부 공고 ✓
  - 남양주 공고 ✓
  - 수원 공고 ✓

[실제 동작]
"경기도 전체" 검색 → location 필드에 "경기도"가 포함된 것만 검색
  - location="경기도" ✓
  - location="의정부" ✗ (매칭 안 됨)
  - location="남양주시" ✗ (매칭 안 됨)
```

### 영향 범위
- **17개 광역시도** 모두 동일한 문제 발생 가능
- 특히 `서울`, `경기`, `인천`, `강원` 크롤러에서 기초자치단체 단위로 저장하므로 문제 심각

**Step 2: 검색 로직 수정 (`queries.ts`)**

```typescript
import { PROVINCE_TO_CITIES } from '@/lib/constants/regionHierarchy';

// 지역 필터 적용 부분 수정
if (hasFilterValue(filters.region, DEFAULT_REGION)) {
  const regionName = filters.region.replace(' 전체', '').replace('도', '');

  // 광역시도 전체 검색인 경우
  if (filters.region.includes('전체') && PROVINCE_TO_CITIES[regionName]) {
    const cities = PROVINCE_TO_CITIES[regionName];
    // 광역시도명 + 모든 기초자치단체명 OR 조건
    const patterns = [regionName, ...cities].map(loc => `location.ilike.%${loc}%`);
    query = query.or(patterns.join(','));
  } else {
    // 기존 로직 (특정 기초자치단체 검색)
    const regionPattern = buildIlikePattern(filters.region);
    query = query.or(`location.ilike.${regionPattern},location.ilike.%${filters.region}시%`);
  }
}
```

### 방안 2: 크롤러에서 광역시도 정보 함께 저장 (보조)

#### 개념
- 크롤러가 location 저장 시 `광역시도|기초자치단체` 형식으로 저장
- 예: `경기|의정부`, `서울|강남구`, `강원|춘천시`

#### 장점
- 검색 시 광역시도만으로 필터링 가능

#### 단점

## 구현 우선순위

### Phase 1: 긴급 패치 (방안 1 단순화)
1. `regionHierarchy.ts` 상수 파일 생성
2. `queries.ts` 검색 로직에 광역시도 → 기초자치단체 확장 적용
3. 기존 필터 옵션에 광역시도 전체 옵션 추가

### Phase 2: UI/UX 개선
1. 지역 필터 드롭다운을 계층형으로 변경
2. 광역시도 선택 → 기초자치단체 목록 표시
3. 검색 결과에 지역 정보 명확히 표시

### Phase 3: 데이터 정규화 (선택)
1. `job_postings` 테이블에 `province_code`, `city_code` 컬럼 추가
2. 기존 데이터 마이그레이션
3. 크롤러 수정하여 정규화된 지역 코드 저장
